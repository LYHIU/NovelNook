**创建日期**: 2026-06-11

**输入**: 用户描述："我要做一个个人书架管理app。UI设计要有审美。用户选择指定小说平台，输入书名或加上作者，检索到书籍信息，然后加入自己的书架，可以标记和做介绍的总笔记（内容用户自定义）。"

## Clarifications

### Session 2026-06-11

- Q: 数据导出功能的优先级和范围？ → A: 纳入 P2 用户故事，支持一键导出书架全部数据为 JSON 文件
- Q: 是否需要支持导入之前导出的 JSON 文件来恢复书架数据？ → A: 纳入 P2，支持从 JSON 文件导入恢复书架（合并到现有数据，不覆盖）
- Q: 非晋江平台（长佩、番茄、起点）是否需要在 MVP 阶段就支持在线书目检索？ → A: MVP 仅晋江支持在线检索，其他平台提供官方搜索链接引导用户手动确认后录入，后续版本扩展
- Q: 离线（无网络）时，哪些功能必须可用？ → A: 离线可浏览书架、筛选、查看/编辑笔记（纯本地操作）；在线搜索时提示"当前无网络"

## 用户场景与测试 *(必填)*

### 用户故事 1 — 按平台搜索书籍（优先级: P1）

用户打开应用后，选择一个小说平台（如晋江文学城），输入小说名称（可选填作者名缩小范围），点击搜索。系统展示匹配的书目信息（书名、作者、平台、链接），用户可浏览搜索结果。

**为何此优先级**: 搜索是用户使用书架的第一步——必须先找到书，才能加入书架管理。没有搜索能力，书架只能是纯手动录入工具。

**独立测试**: 可通过选择一个平台、输入关键词、验证搜索结果列表来完整测试，无需依赖书架或笔记功能。

**验收场景**:

1. **Given** 用户打开应用，**When** 用户选择"晋江文学城"并输入"造神"点击搜索，**Then** 系统展示包含"造神"的书目列表，每项含书名、作者、平台来源、原站链接。
2. **Given** 用户已在晋江平台输入书名，**When** 用户同时输入作者名"星海浮萍"，**Then** 搜索结果仅展示该作者的作品。
3. **Given** 用户选择非晋江平台，**When** 用户输入书名搜索，**Then** 系统生成该平台的官方搜索入口链接，引导用户自行确认。
4. **Given** 用户未输入书名直接点搜索，**When** 搜索触发，**Then** 系统提示"请输入小说名"。

---

### 用户故事 2 — 将书籍加入个人书架（优先级: P2）

用户从搜索结果中确认一本书后，可一键将其加入个人书架。加入时自动带入书名、作者、平台、作品ID、原站链接等基本信息。用户也可手动补充字数、连载状态等信息。

**为何此优先级**: 书架是产品的核心价值——搜索只是发现手段，存储和管理才是用户打开这个 app 的目的。

**独立测试**: 可从搜索结果中选择一本书加入书架，然后在书架列表中验证该书出现且信息完整。

**验收场景**:

1. **Given** 用户在搜索结果中看到目标书籍，**When** 用户点击"加入书架"，**Then** 该书出现在个人书架列表中，含书名、作者、平台、原站链接。
2. **Given** 用户想添加一本未搜到的书，**When** 用户手动填写书名、作者等信息并保存，**Then** 该书加入书架，信息完整。
3. **Given** 用户粘贴晋江作品链接（如 `https://www.jjwxc.net/onebook.php?novelid=1234567`），**When** 系统解析链接，**Then** 自动提取作品ID并填入表单。
4. **Given** 书架中已有同名书籍，**When** 用户再次加入同一本书，**Then** 系统更新已有记录而非创建重复条目。
5. **Given** 用户书架中有若干书籍，**When** 用户点击"导出书架数据"，**Then** 系统生成包含全部书籍信息及阅读记录的 JSON 文件并触发下载。
6. **Given** 用户手头有一份之前导出的书架 JSON 文件，**When** 用户选择该文件导入，**Then** 文件中的书籍合并到当前书架（已存在的书籍按 novelid 去重，保留最新记录），不覆盖用户未导出的数据。

---

### 用户故事 3 — 标记阅读状态与撰写个人笔记（优先级: P3）

用户对书架中的每本书可以设置阅读状态（想看/在看/看完/暂搁/弃文/想二刷），并撰写完全由用户自定义的私人笔记（如：好看的点、雷点、弃文原因、读到哪里了）。还可以给书打上个性化标签（如"强剧情""群像""雷点待查"）。

**为何此优先级**: 标记和笔记是书架管理的深度价值——让书架从"书单"升级为"阅读日记"。但即使没有笔记，书架本身已有基本管理价值。

**独立测试**: 可在已有书架数据的基础上，对任意一本书设置状态、写笔记、加标签，然后刷新验证信息持久化。

**验收场景**:

1. **Given** 书架中有一本书，**When** 用户将其状态设为"看完"，**Then** 书架列表中该书显示"看完"标记。
2. **Given** 用户在某本书的笔记区写了一段文字，**When** 用户保存后重新打开该书，**Then** 笔记内容完整保留。
3. **Given** 用户给书添加标签"群像, 强剧情"，**When** 用户保存后查看该书，**Then** 标签以独立标签样式展示。
4. **Given** 用户通过筛选器选择"想看"，**When** 书架列表刷新，**Then** 仅展示状态为"想看"的书籍。

---

### 边界情况

- 用户输入超长书名（>100字符）时，搜索请求应正常处理，不崩溃。
- 搜索结果为空时，展示友好的空状态提示，并引导用户使用官方搜索链接确认。
- 书架为空时，展示引导文案，提示用户从搜索开始添加第一本书。
- 后端搜索代理不可用时（网络故障/服务未启动），前端展示明确错误提示，不白屏。
- 用户快速连续点击"加入书架"时，应防止重复添加同一本书。
- 标签数量超过12个时，自动截断保留前12个。
- 导入的 JSON 文件格式不正确或内容损坏时，系统应提示"文件格式不正确"，不破坏现有书架数据。
- 离线状态下打开应用，书架浏览和笔记编辑应正常工作；点击在线搜索时提示"当前无网络，请检查网络连接"。

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系统 MUST 提供平台选择器，至少支持晋江文学城、长佩文学、番茄小说、起点中文网四个平台。
- **FR-002**: 系统 MUST 允许用户输入书名关键词进行搜索，可选填作者名进一步筛选。
- **FR-003**: 系统 MUST 对晋江文学城平台提供在线书目检索能力（通过后端代理），返回书名、作者、作品ID、原站链接、简介摘要。
- **FR-004**: 系统 MUST 对未接入在线检索的平台，生成对应平台的官方搜索入口链接。
- **FR-005**: 系统 MUST 优先检索用户本地书架中的已有记录，命中时直接展示。
- **FR-006**: 用户 MUST 能够将搜索结果中的书籍一键加入个人书架，自动填入已知字段。
- **FR-007**: 用户 MUST 能够手动输入/编辑书籍信息（书名、作者、平台、字数、连载状态、简介等）。
- **FR-008**: 系统 MUST 支持解析晋江作品页链接，自动提取 novelid 并填入表单。
- **FR-009**: 系统 MUST 支持 6 种阅读状态标记：想看、在看、看完、暂搁、弃文、想二刷。
- **FR-010**: 用户 MUST 能够为每本书撰写私人笔记（自由文本，至少支持 1000 字）。
- **FR-011**: 用户 MUST 能够为每本书添加个性化标签（逗号分隔，最多 12 个）。
- **FR-012**: 系统 MUST 支持按关键词和阅读状态筛选书架中的书籍。
- **FR-013**: 用户 MUST 能够复制书籍的原站链接，快速跳回官方平台。
- **FR-014**: 系统 MUST 支持删除书架中的书籍记录，删除前需用户确认。
- **FR-015**: 所有书架数据 MUST 存储在用户本地设备，不经过服务端持久化。
- **FR-016**: 用户 MUST 能够一键导出书架全部数据（含书籍信息、阅读状态、笔记、标签）为 JSON 格式文件。
- **FR-017**: 用户 MUST 能够导入之前导出的 JSON 文件，将其中书籍合并到当前书架（按 novelid 去重，保留最新记录），不覆盖已有数据。
- **FR-018**: 系统 MUST 在离线状态下支持书架浏览、筛选、查看/编辑笔记和标签等纯本地操作；在线搜索功能不可用时 MUST 显示明确提示而非白屏或假死。

### 关键实体

- **书籍（Book）**: 书名、作者、所属平台、作品ID、原站链接、字数、连载状态、简介摘要。一条书籍记录代表用户书架中的一个条目。
- **阅读记录（ReadingRecord）**: 与书籍一一关联，包含阅读状态（6选1）、私人笔记文本、标签列表（最多12个）。记录用户对该书的个人化信息。
- **搜索请求（SearchQuery）**: 平台标识、书名关键词、可选的作者名。代表一次跨平台或本地书架的检索操作。

## 成功标准 *(必填)*

### 可衡量成果

- **SC-001**: 用户从打开 app 到成功将一本书加入书架的全流程，在 1 分钟内可完成。
- **SC-002**: 搜索结果（在线检索）在网络正常时 10 秒内返回结果或明确提示。
- **SC-003**: 书架筛选操作的响应感知为即时（用户主观感受无延迟）。
- **SC-004**: 用户首次使用时无需阅读帮助文档即可完成"搜索→加入书架"的核心操作。
- **SC-005**: 书架数据在应用关闭后重新打开时 100% 保留不丢失。

## 假设

- 用户具备基本的智能手机操作能力，能理解"平台""搜索""书架"等概念。
- 用户有稳定的网络连接（用于在线搜索功能）。
- 晋江文学城的在线搜索接口（search_ajax.php）保持可用；如不可用，Baidu 搜索引擎回退通道作为补充。
- 用户主要使用 Android 手机浏览器或微信小程序访问应用，屏幕宽度在 360px–414px 之间。
- 当前版本仅支持中文书籍和中文界面，不考虑国际化。
- 用户理解本应用不提供小说在线阅读功能，仅管理书目和个人笔记。
- MVP 阶段仅晋江文学城支持在线书目检索；长佩、番茄、起点等其他平台的在线检索在后续版本中逐步接入。

## 合规自评

参照项目宪法「合规与伦理边界」章节：

- 本功能不涉及抓取小说正文、付费章节、章节目录。 ✅
- 在线搜索仅获取公开书目元数据（书名、作者、作品ID、链接）。 ✅
- 搜索结果标注来源平台，引导用户回到官方平台。 ✅
- 不缓存平台封面图、简介文案。 ✅
- 所有用户笔记和书架数据存储在本地。 ✅
- 不伪装成平台官方客户端。 ✅
